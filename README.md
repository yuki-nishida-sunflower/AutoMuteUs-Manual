# AutoMuteUs-Manual 🚀
**AutoMuteUs が動かない・導入が難しい時のための Among Us 手動ミュート制御ツール**

本プロジェクトは、本家 `AutoMuteUs` や `Galactus` などの Bot がダウンしている際や、セルフホスト環境の構築が難しい方向けの**軽量な代用ツール**です。

## ❓ こんな時に便利です
- 「AutoMuteUs がオンラインにならない」
- 「本家 Bot の反応が遅い、または設定が複雑すぎる」
- 「特定のメンバーだけ手動で素早く制御したい」
- 「複雑な Docker 設定なしで、EXEを叩くだけで動かしたい」

## 💡 AutoMuteUs の代用として
本ツールは `discord.py` を使用し、非常に軽量に動作します 。
「Bot がサーバーにいない」「コマンドに反応しない」といったトラブル時のバックアップとして、デスクトップに置いておくだけですぐに使用可能です。

## ✨ 特徴
- **直感的な GUI**: メンバーを一覧表示し、生存/死亡をワンクリックで切り替え。
- **差分最適化制御**: 現在のステータスを監視し、変更が必要なメンバーにのみ命令を送信することで通信負荷を軽減。
- **安心の操作ロック**: ミュート実行中はボタンをロックし、誤操作や多重送信を防止。
- **自動設定**: 初回起動時に設定ファイルのテンプレートを自動生成。

## 🛠 導入・設定ガイド

本ツールを使用するには、Discord Bot の作成と設定ファイルの編集が必要です。

### 1. Discord Bot の作成とトークン取得
1. [Discord Developer Portal](https://discord.com/developers/applications) にアクセスします。
2. **「New Application」** を押し、任意の名前で作成します。
3. 左メニューの **「Bot」** を選び、**「Reset Token」** を押して表示された文字列をコピーします。
   - ⚠️ **これが `BOT_TOKEN` です。公開しないでください。**
4. 同じ画面（Bot）を下へスクロールし、**「Privileged Gateway Intents」** セクションにある以下の3つをすべて **ON** にして保存します。
   - `PRESENCE INTENT`
   - `SERVER MEMBERS INTENT`
   - `MESSAGE CONTENT INTENT`

### 2. 各種 ID の取得（開発者モード）
1. Discordアプリの **「ユーザー設定」>「詳細設定」** から **「開発者モード」** を **ON** にします。
2. **サーバーID (`GUILD_ID`)**: サーバー名を右クリックして「サーバーIDをコピー」します。
3. **チャンネルID (`VOICE_CHANNEL_ID`)**: 使用するボイスチャンネルを右クリックして「チャンネルIDをコピー」します。

### 3. Bot の招待
1. Developer Portal の **「OAuth2」>「URL Generator」** を開きます。
2. **Scopes** で `bot` と `applications.commands` を選択。
3. **Bot Permissions** で以下にチェックを入れます。
   - `Mute Members`（メンバーをミュート）
   - `Deafen Members`（メンバーのスピーカーをミュート）
   - `Move Members`（メンバーを移動）
4. 生成された URL をブラウザで開き、自分のサーバーに Bot を招待します。

### 4. 実行と設定
1. [Releases](https://github.com/yuki-nishida-sunflower/AutoMuteUs-Manual/releases/tag/v1.2.0) から最新の `AutoMuteUs-Manual.exe` をダウンロードします。
2. 実行すると同じフォルダに `local.config` が生成されます。
3. `local.config` をメモ帳などで開き、取得した値を貼り付けて保存します。
   ```ini
   [DISCORD]
   BOT_TOKEN = （ここにトークン）
   GUILD_ID = （ここにサーバーID）
   VOICE_CHANNEL_ID = （ここにチャンネルID）
4. 再度アプリを起動し、「① Config読み込み」→「② Discord接続」の順に進めてください。

## 🎮 操作方法（ゲーム中の使い方）

設定が完了したら、以下の手順でゲーム中のミュート管理を行います。

### Step 1: メンバーの読み込み
ゲームを始める前に、参加者全員が指定のボイスチャンネル（VC）にいることを確認してください。

1.  アプリの **「メンバー更新」** ボタンを押します。
2.  その VC にいるメンバー全員のボタンが生成されます。

### Step 2: 生存/死亡の管理（ゲーム進行中）
議論フェーズが終わってゲーム（タスクフェーズ）が始まる前、あるいは議論中に追放された時などに操作します。

1.  死亡したメンバーのボタンをクリックします。
2.  ボタンが **「赤色」** になり、死亡状態としてマークされます。
    - もう一度クリックすると「白色（生存）」に戻ります。
[Image: image_9_annotated.png] * 画像はそのうち
> 💡 この時点ではまだ Discord のミュートは変わりません。アプリ内でステータスを管理している状態です。

### Step 3: フェーズ切り替え（ミュート実行）
画面下部のフェーズボタンを押すことで、Step 2 で設定したステータスに基づいて、全員のミュート状態を瞬時に切り替えます。

- **「待機」**: 全員のミュート・スピーカーミュートを解除します（ゲーム前後の雑談用）。
- **「タスク」**: 生存者（白）のみをミュート＆スピーカーミュート（強ミュート）します。死亡者（赤）はそのまま会話可能です。
- **「会議」**: 死亡者（赤）のみをマイクミュートにします。生存者（白）は会話可能です。

### 使用上の注意点
ご利用になるうえで、導入者あるいは参加者が気を付けなければいけないこと

- 会議フェーズ、タスクフェーズが始まると自動で切り替えができません。必ず、導入者がそれぞれのボタンを押して切り替える必要があります。（ボタンを押したらゲームに戻ってOK）
- 死亡者が居る場合、後から切り替えることになるので、参加者は会議開始後は若干、喋るのを待った方が良い。

## 🔒 セキュリティとプライバシー

- **トークンの保護**: 本ツールは、トークン情報を `local.config` にのみ保存します。外部サーバーへの送信や GitHub への Push は絶対に行わないでください。
- **必要最低限の権限**: Bot の権限は「ミュート」「スピーカーミュート」「メンバーの移動（必要な場合）」のみを要求します。

## 🚀 動作が遅い・動かない場合
- Botの権限を確認: サーバー設定の「ロール」で、本Botのロールを操作したいメンバーのロールよりも上に配置してください。
- ボイスリージョンの確認: サーバー設定で地域が「Japan」になっているか確認してください。
- 管理者実行: Windows の保護機能により動作が制限される場合があるため、右クリックから「管理者として実行」を試してください。

## 🛠️ 開発環境
- Python 3.12
- discord.py
- tkinter (GUI)
- PyInstaller (Build)

### build コマンド
```
PS C:path> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process
PS C:path> .\venv\Scripts\Activate.ps1
(venv) PS C:path> python -m PyInstaller --onefile --noconsole --name "AutoMuteUs-Manual" --icon="icon.ico" main.py
# --noconsole を外してビルド（ログ確認用）
(venv) PS C:path> python -m PyInstaller --onefile --console --name "AutoMuteUs-Manual-console" --icon="icon.ico" main.py
```

### 🎨 アイコンのカスタマイズ
独自のアイコンを使用したい場合は、以下の手順で行ってください。
1. `icon.ico` という名前のアイコンファイルをプロジェクトルートに配置します。
2. ビルド時に `--icon="icon.ico"` オプションを付与して実行します。
